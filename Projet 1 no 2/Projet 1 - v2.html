Entreprise Web

Des experts en la matière!

Aperçu

Votre partenaire et vous prenez la décision de démarrer une entreprise de Design Web pour vous aidez à payer vos frais de scolarité.

Comme vous n'avez jamais travaillé ensemble, vous convenez qu'il serait bien de créer un premier site web ensemble pour mesurer la qualité de celui-ci.

Vous faites quelques recherches et vous constatez que la meilleure façon de procéder est de reproduire un site web existant. Au boulot!

Prérequis

Avant de commencer, vous devez choisir un site web que vous allez reproduire parmis la liste suivante.

  • aThemes
  • ThemeIsle
  • ZakraTheme
  • GeneratePress
  • ThemeGrill

Vous devrez faire:

  • Au minimum 3 pages différentes.
  • Au moins une page contenant un formulaire.

Notes

  • ATTENTION! Vous devez reproduire le site choisi, et non vous servir du produit afin de créer un site différent.
  • Vous êtes évalué sur la fidélité de votre représentation, il est primordial que tout le contenu des pages modèles choisies soit présent dans votre remise.
  • À titre indicatif, un exemple de grandeur adéquate pour ce projet est: les pages home, login, premium du site GeneratePress.
  • Si vous avez des doutes sur la grandeur de votre projet, confirmez votre choix auprès du correcteur avant de débuter (David Génois).

Remise

Vous devez remettre:

  • Le nom du site choisi.
  • Trois pages HTML. (voir détails)
  • Une ou plusieurs feuilles de styles CSS. (voir détails)
  • Un petit rapport. (voir détails)
  • Un fichier zip contenant le dossier, le travail doit fonctionner directement en ouvrant les fichiers HTML une fois le dossier dézipper, et ce, peu importe l'emplacement dans lequel le dossier ce trouve.

Code

Tout votre code doit être écrit à la main et vos pages web doivent être statiques.

Donc, vous ne pouvez pas utiliser:

  • Des librairies de styles CSS. (L'utilisation de FontAwesome pour des icônes est OK)
  • Des moteurs de template.
  • JavaScript.

HTML & CSS

3 pages HTML avec au moins un fichier CSS.

Essayez d'organiser votre CSS pour qu'il n'y ai pas de duplication de code. Des duplications de code excessive entraineront des pertes de points. Tout votre style devrait être contenu dans des fichiers CSS et non dans des attributs/balises style à même votre fichier HTML

Vous êtes encouragés à:

  • Avoir une bonne structure de fichier et de bien les nommer. (Exemple: séparer le CSS/HTML et les images, etc.)
  • Ne pas inclure des balises SVG directement dans le HTML, plutôt relier un fichier .svg (dans un cas pour lequel il y a un SVG dans la page)
  • Vérifier que votre page web fonctionne en format desktop pour différentes tailles d'écrans et résolution.

Notes

  • Vous pouvez utiliser flexbox.
  • Votre site web doit être disponible que dans un format desktop, donc pas de version tablette ou téléphone..
  • La validation avec W3C tel que vu en classe est un bon moyen de vérifier ces erreurs. Des fichiers sans erreurs de validation risques d'être mieux notés
  • ATTENTION! Même si vous pouvez voir le code du site pour vous en inspirer, assurez-vous d'être conforme aux politiques sur le plagiat. Il est inacceptable de simplement copier-coller du code.

Formulaire(s)

Le(s) formulaire(s) n'a/ont pas à envoyer (ses/leurs) d'informations.

Reproduction

Vous devez reproduire (au meilleur de vos compétences):

  • Les différents espacements.
  • Les couleurs.
  • Les polices (tailles et épaisseurs).
  • Effets CSS (hover).

Les animations JavaScript (ou trop complexes) ne doivent pas être reproduites.

Validité

Votre code HTML5 et CSS3 doit être valide.

Rapport

Le rapport doit faire entre 10 et 20 lignes (environ 1/2 à une page maximum). Le rapport est surtout pour donner du context entourant votre travail au correcteur.

Le rapport doit contenir :

  • Les noms complets tel qu'écrit sur Studium et le matricule de chaque coéquipier.
  • L'adresse vers le site web original.
  • Une description du travail réalisé par chaque étudiant.
  • Les difficultés que vous avez rencontrées.
  • Une courte réflexion de quelques ligne sur votre travail et votre niveau de satisfaction.
  • Tout autre commentaire ou explication pertinant pour le correcteur.

Notes

  • Autre que pour fournir vos matricules, ce rapport vous donne la possibilité de faire comprendre votre situation au correcteur, de partager vos interprétations ou prises de décisions significatives, ainsi que de partager votre opinion sur ce projet. Le contenu ne sera pas évalué, mais servira plutôt à améliorer ce projet dans le futur.

Critères de correction

Outre les points mentionnés ci-dessus, le travail sera corrigé en prenant compte des critères suivants:

  • Les consignes ont été respectées.
  • Le résultat final est fidèle au modèle et est visuellement plaisant.
  • Les connaissances vues en classe sont appliquées dans le projet.
